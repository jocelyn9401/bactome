def genbank.displayFeatureTypes (   gbfile = 'DSM6083.gb')

Function to read a Genbank file and prints out the number of each feature for each Genbank record in the file.

Usage: 

python genbank.py displayfeaturetypes --gbfile=<Genbank 
file path>

The format of the output is 

<feature type> : <count>

where count is the number of features for each feature type.

Parameters
gbfilestring: file path of the Genbank file

Function to print out the base position(s) or base range for a specific feature type in a specific Genbank record within the given Genbank file.

Usage 1: Prints the range of bases for a required feature 

python genbank.py featuring --feature=<feature type> 
--RecordIndex=<Record number> --gbfile=<Genbank file path>

Usage 2: Prints out individual base position for a required feature 

python genbank.py featuring --feature=<feature type> 
--RecordIndex=<record number> --gbfile=<Genbank file path> 
--expand=False --resolution=<resolution>

where

  • record number is the relative Genbank record number in the given Genbank file where 0 is the first record and 1 is the secord record, and so on
  • resolution is to control the resolution of printouts. For example, resolution of 1 will print out every base position for a given feature type; which will generate a long printout. Increasing the resolution will reduce the volume of printout but may not accurately pin-point the end base position of the feature

Usage 1 will generate the following output: 

<feature type> : <starting base position> : <ending 
base position>

Usage 2 will generate the following output: 

<feature type> : <base position>
Parameters
featureString: Type of feature
RecordIndexInteger: Relative Genbank record number in the given Genbank file where 0 is the first record and 1 is the secord record, and so on
RecordIDString: Unique name (identifying IDs) for the specific Genbank record
gbfilestring: file path of the Genbank file
expandBoolean: Flag to define whether to expand from start/end base positions to indivisual base positions
resolutionInteger: Control the resolution of printouts. For example, resolution of 1 will print out every base position for a given feature type; which will generate a long printout. Increasing the resolution will reduce the volume of printout but may not accurately pin-point the end base position of the feature
